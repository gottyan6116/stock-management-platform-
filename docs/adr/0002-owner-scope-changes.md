# ADR 0002: オーナー指示によるスコープ変更（ポートフォリオ・投資信託・サイドバー5項目）

## 変更対象
- 左サイドバー項目数（設計書7.1「3項目のみ」→ 5項目）
- 保有資産（positions）の手入力・評価額表示（設計書4.2でMVP外 / 4.3でPhase 2候補）
- 投資信託の検索・表示（設計書11.1でMVP検索対象外）

## 変更理由
2026-07-07、プロダクトオーナー（ユーザー本人）からの明示的な指示:
1. サイドバーに「投資信託」を追加し準ずるページを作成する
2. 「お気に入り銘柄」の上に「ポートフォリオ」を追加し、保有資産の閲覧・手入力ができるページを作成する

## 代替案
1. 設計書を盾に実装を拒否し、Phase 8完了後に対応 → オーナーの現在の意思に反する
2. 別アプリとして分離 → 個人利用アプリで分離コストに見合わない
3. 本体に追加し、ADRで逸脱を記録（採用）

## 採用案
- サイドバー: ポートフォリオ / お気に入り銘柄 / 日本株 / 米国株 / 投資信託 の5項目
- `positions`テーブル（RLS付き、手入力のみ）と `/api/positions` を新設
- Yahoo検索のMUTUALFUNDを `instrument_type='fund'` として許可
- 検索結果はJP/US市場のシンボルのみに制限（`.T`または無サフィックスのみ。SPCX34.SA等の他国市場を除外）

## UX・データ・コストへの影響
- ポートフォリオ評価はリクエスト時にProviderからQuote取得（保有銘柄数が少ない前提。多くなればPhase 7の同期データに切替）
- 設計書5.2の条件（保有情報が1件以上ある場合のみ評価額表示）は満たされる

## 後方互換性
- 既存テーブルは変更なし（instrumentsのcheck制約にfundを追加するのみ）
- migration 0002はSupabase SQL Editorで手動適用が必要

## ロールバック方法
- サイドバーから2項目を削除し、/portfolio・/funds ルートと /api/positions を削除
- `drop table public.positions;` とcheck制約の巻き戻し
